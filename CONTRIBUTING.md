## 📋 The contribution process

```
1. Open an issue  →  2. Wait for authorization  →  3. Implement  →  4. Submit PR  →  5. Review & merge
```

### Step 1 — Open an issue first
Before writing a single line of code, [open an issue](https://github.com/prashantkeshr/Dhurta/issues/new) describing:
- What you want to add or fix
- Why it matters for privacy / usability / performance
- Your rough implementation approach

### Step 2 — Wait for Owner authorization
The Owner will respond with one of:
- ✅ **Authorized** — go ahead; scope and constraints will be noted
- 💬 **Discuss** — let's refine the approach first
- ⏸️ **Deferred** — good idea, but not right now
- ❌ **Declined** — with an explanation

Do not begin implementation until you receive an explicit ✅.

### Step 3 — Implement within authorized scope
- Follow the existing code style (TypeScript, React, Electron)
- Keep changes minimal and focused — no scope creep
- Test your change on Windows (primary), macOS, and Linux if possible
- No new external dependencies without discussion

### Step 4 — Submit a pull request
- Reference the authorizing issue in your PR description
- Summarize what changed and why
- All PRs are *proposals* until merged by the Owner

### Step 5 — Review & merge
The Owner reviews all PRs. Accepted contributions are merged under the project's proprietary license.

## 🐛 Reporting bugs (no authorization needed)

Bug reports are **always welcome**, no authorization required. Use the [issue tracker](https://github.com/prashantkeshr/Dhurta/issues/new) and include:

- Your OS and version (Windows 11, macOS 15, Ubuntu 24.04, etc.)
- Dhurta version (`v1.0.8.0`)
- Exact steps to reproduce
- What you expected vs. what happened
- Screenshots or logs if available

## 🔒 Security disclosures

Found a security vulnerability? **Please do NOT open a public issue.**

Report it privately via email to **Dhurta.inc** (contact the Owner via GitHub profile). Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact

We commit to:
- Acknowledging your report within 72 hours
- Keeping you informed as we investigate and fix
- Crediting you publicly (if you wish) when the fix ships
